# System Design from Zero

An interactive, single-file learning guide that teaches system design from absolute basics
("what *is* a server?") all the way to designing distributed systems end-to-end — written for
working engineers who know the words (Docker, Kubernetes, load balancer, instance) but not yet
what they mean, when to use them, or why.

**▶ Live site:** https://shawon39.github.io/system-design-from-zero/

## What's inside

- **12 sections**, ordered foundational → advanced. Each concept explains **what it is, why it
  exists, how it works, and when to use it vs. not** — grounded in a concrete real-world scenario,
  with every piece of jargon defined the first time it appears.
- **67 Mermaid diagrams** for every architectural concept.
- **Fact-checked before writing.** Every claim was verified against primary sources (RFCs, MDN,
  AWS, Google SRE, Kubernetes, Docker, PostgreSQL, Kafka, OWASP, Martin Fowler, *Designing
  Data-Intensive Applications*, and more), with sources cited inline and per section.

### Curriculum

1. Foundations: one request, one program & the vocabulary
2. Networking & cloud topology: how a request travels
3. Scaling out: load balancing & statelessness
4. The data layer: storage, durability, replication & sharding
5. Making it fast: caching & CDNs
6. Distributed reality: clocks, consistency, CAP & PACELC
7. Agreement: quorums & consensus
8. Application correctness: idempotency, delivery & transactions
9. Communication: async messaging, APIs & protocols
10. Architecture, resilience & the deployment substrate (Docker & Kubernetes)
11. Operating it: observability, autoscaling, reliability & security
12. Capstones: putting it all together (URL shortener, news feed, chat, video, ride-sharing)

## Using it

It's a **single self-contained `index.html`** — just open it in a browser (or visit the live site).
Features: a collapsible left-sidebar accordion, light/dark themes, responsive layout, and
shareable deep links to any section or topic via the URL hash.

> Diagrams render via Mermaid loaded from a pinned CDN, so an internet connection is needed for the
> diagrams to draw.
